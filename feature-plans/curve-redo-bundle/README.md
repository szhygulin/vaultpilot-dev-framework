# Curve-redo experiment — Phase 2 bundle

Self-contained kit for running the curve-redo experiment described in [`how-the-proposed-plan-bubbly-peach.md`](https://github.com/szhygulin/claude-md-global) (the approved plan; copy in operator's `~/.claude/plans/`). Replaces #179's K=13 result with a falsifiable 0–100 quality metric: hidden-test pass rate (B, 0–50) + blinded reasoning judge (A, 0–50), summed for `implement` cells, doubled-A for `pushback`.

Coding agent: **Sonnet 4.6** throughout (`--model claude-sonnet-4-6`). Test generator + reasoning judge: **Opus 4.7**.

## Quality metric (recap)

```
quality_per_cell =
   A + B    if decision == "implement" AND test apply succeeded   (range 0..100)
   2 × A    if decision == "pushback"                              (range 0..100)
   0        if decision == "error" / parse failure / apply failure
```

- `A` ∈ 0..50 — Opus K=3 blinded judge median; reads issue body + agent's diff (or pushback comment), strips agent IDs / branch names / replicate hints. Equal weight with B by design.
- `B` ∈ 0..50 — `(passed / 100) × 50` from the issue's 100 hidden tests applied to the agent's worktree-diff

Per-agent quality = mean of cell qualities. Curve sample factor = qmax / quality (anchored at 1.0 for the highest-quality agent). **Factor is scale-invariant** under any quality rescale (a 50% drop in quality yields factor 2.0 whether quality maxes at 100 or 1000), so the 0-200 → 0-100 quality rescale is invisible to the curve fit and to runtime consumers.

Same regression machinery (linear-log default) feeds `src/util/contextCostCurve.ts`. At runtime, `rangeNormalize` (in `contextCostCurve.ts:226`) maps each curve's raw factors onto a common `[1, 2]` scale before the weighted composite, so accuracy and token-cost contribute proportionally to their stated weights regardless of their raw maxFactor values. The combined proposal JSON emits both `samples` (raw) and `normalizedSamples` (mapped to `[1, 2]`) so the operator can see the comparable shape at synthesis time too.

## Two-leg structure

| Leg | Target repo | Framework | Issues | Cells | Wall | Cost |
|---:|---|---|---:|---:|---:|---:|
| 1 | `szhygulin/vaultpilot-mcp` | vitest | 6 | 108 | ~75 min | ~$120 |
| 2 | `szhygulin/vaultpilot-development-agents` | node-test | 7 | 126 | ~90 min | ~$140 |

Per-leg cost: ~$120–$140 cells (Sonnet @ $1.10/cell) + ~$150 judge (K=3 × Opus @ $0.50/cell). Total experiment: ~$580 cells+judge + ~$15 test generation = **~$595**. Cap: `--max-total-cost-usd 1500` per leg (defense in depth).

Each leg runs on a different machine. Both pull this bundle, each runs its leg, each commits its results to a feature branch, one machine then pulls both branches and runs the combiner + fit.

## Workflow

- Clone of this repo at `$REPO`. `npm ci && npm run build` succeeds.
- For leg 1: clone of `szhygulin/vaultpilot-mcp` at `$HOME/dev/recon-mcp` (or anywhere — only used as a clone source).

- `ANTHROPIC_API_KEY` set.
- `npx tsx --version` works (testRunner default; `npm i -g tsx` if missing).


### Step 3 — leg-specific: dispatch with replay-mode + capture-diff + Sonnet model

Hidden tests are committed in `feature-plans/curve-redo-bundle/curve-redo-tests/<issueId>/`. Per-cell diffs land in `feature-plans/curve-redo-data/diffs-leg<N>/<cellId>.diff`. Cell logs land in `feature-plans/curve-redo-data/logs-leg<N>/`.

`vp-dev research curve-study` already hardcodes CLAUDE.md isolation (`suppressTargetClaudeMd: true`) per [#216](https://github.com/szhygulin/vaultpilot-development-agents/pull/216), so no flag needed for that. **But curve-study itself doesn't yet wire `--model` / `--replay-base-sha` / `--capture-diff-path` into the cell spawn.** For the curve-redo experiment, dispatch via a thin shell loop calling `vp-dev spawn` directly (one cell at a time, per-trim parallelism via `&`):

`--capture-diff-path` defaults its diff base to the worktree's HEAD at spawn-start when `--replay-base-sha` isn't set, so leg-1 open-issue cells correctly capture committed agent work without operator-side wiring. Leg-2 closed-issue cells still pass `--replay-base-sha` for the rollback step.

```bash
# Leg 1 dispatch script (run on machine 1)
mkdir -p feature-plans/curve-redo-data/{logs-leg1,diffs-leg1,scores-leg1}

LEG=1
TARGET_REPO=szhygulin/vaultpilot-mcp
jq -r '.issues[] | select(.leg == 1) | "\(.issueId)\t\(.state)\t\(.baseSha // "")"' \
  feature-plans/curve-redo-bundle/corpus.json > /tmp/leg1-issues.tsv

jq -r '.[] | select(.clonePath | contains("vaultpilot-mcp")) | "\(.devAgentId)\t\(.clonePath)"' \
  feature-plans/issue-179-leg2-bundle/agents-spec-phase3.json > /tmp/leg1-trims.tsv

while IFS=$'\t' read -r issueId state baseSha; do
  while IFS=$'\t' read -r agentId clonePath; do
    cellId="${agentId}-${issueId}"
    extra=""
    [ -n "$baseSha" ] && extra="--replay-base-sha $baseSha"
    node dist/bin/vp-dev.js spawn \
      --agent "$agentId" \
      --issue "$issueId" \
      --target-repo "$TARGET_REPO" \
      --target-repo-path "$clonePath" \
      --dry-run --skip-summary \
      --allow-closed-issue --issue-body-only --no-target-claude-md \
      --model claude-sonnet-4-6 \
      --capture-diff-path "feature-plans/curve-redo-data/diffs-leg${LEG}/${cellId}.diff" \
      $extra \
      > "feature-plans/curve-redo-data/logs-leg${LEG}/curveStudy-${cellId}.log" 2>&1 &
    # Throttle: max 4 concurrent
    while [ $(jobs -r | wc -l) -ge 4 ]; do sleep 1; done
  done < /tmp/leg1-trims.tsv
  wait
done < /tmp/leg1-issues.tsv

ls feature-plans/curve-redo-data/logs-leg1/curveStudy-*.log | wc -l   # expect 108
```

(Leg 2 is the symmetric script — `LEG=2`, `TARGET_REPO=szhygulin/vaultpilot-development-agents`, leg-2 corpus filter, leg-2 trims-tsv.)

### Step 4 — leg-specific: score each cell (testRunner + reasoningJudge)

```bash
LEG=1
TARGET_REPO=szhygulin/vaultpilot-mcp
mkdir -p feature-plans/curve-redo-data/scores-leg${LEG}

# For each captured diff, run hidden tests + reasoning judge
for diff in feature-plans/curve-redo-data/diffs-leg${LEG}/*.diff; do
  cellId=$(basename "$diff" .diff)
  # cellId = <agentId>-<issueId>
  agentId=$(echo "$cellId" | sed 's/-[0-9]*$//')
  issueId=$(echo "$cellId" | sed 's/.*-//')
  framework=$(jq -r --arg id "$issueId" '.issues[] | select(.issueId == ($id | tonumber)) | .framework' \
    feature-plans/curve-redo-bundle/corpus.json)
  decision=$(jq -r '.envelope.decision // "error"' "feature-plans/curve-redo-data/logs-leg${LEG}/curveStudy-${cellId}.log" \
    | tail -1)
  baseSha=$(jq -r --arg id "$issueId" '.issues[] | select(.issueId == ($id | tonumber)) | .baseSha // ""' \
    feature-plans/curve-redo-bundle/corpus.json)
  testsDestRelDir=$(jq -r --arg id "$issueId" '.issues[] | select(.issueId == ($id | tonumber)) | .testsDestRelDir // ""' \
    feature-plans/curve-redo-bundle/corpus.json)

  # Fresh clone at baseSha (or origin/main if open issue). Symlink the
  # source clone's `node_modules/` so vitest can find its config-time
  # imports (`vitest/config`) without re-running `npm ci` per cell. The
  # source clones must have been `npm ci`'d once (Step 2.5 below).
  cellClone="/tmp/curve-redo-runtest-${cellId}"
  rm -rf "$cellClone"
  srcClone="/tmp/curve-redo-clones-leg${LEG}/${agentId}-vaultpilot-mcp"
  git clone --no-local "$srcClone" "$cellClone"
  ln -s "$srcClone/node_modules" "$cellClone/node_modules"
  [ -n "$baseSha" ] && (cd "$cellClone" && git reset --hard "$baseSha")

  # Test pass rate (B). When the corpus pins testsDestRelDir, the hidden
  # tests use sibling imports (`./<x>.js`) and need to live next to the impl
  # in `src/agent/` / `src/state/` / etc. Default placement is the
  # `curve-redo-hidden-tests/` sandbox subdir.
  destFlag=()
  [ -n "$testsDestRelDir" ] && destFlag=(--tests-dest-rel-dir "$testsDestRelDir")
  node dist/bin/vp-dev.js research run-tests \
    --diff-path "$diff" \
    --tests-dir "feature-plans/curve-redo-bundle/curve-redo-tests/${issueId}" \
    --clone-dir "$cellClone" \
    --framework "$framework" \
    "${destFlag[@]}" \
    --out "feature-plans/curve-redo-data/scores-leg${LEG}/${cellId}-tests.json"

  # Reasoning judge (A)
  if [ "$decision" = "implement" ]; then
    node dist/bin/vp-dev.js research grade-reasoning \
      --issue "$issueId" --target-repo "$TARGET_REPO" \
      --decision implement --diff-path "$diff" \
      --out "feature-plans/curve-redo-data/scores-leg${LEG}/${cellId}-judge.json"
  elif [ "$decision" = "pushback" ]; then
    # Extract pushback reason from envelope
    jq -r '.envelope.reason // ""' "feature-plans/curve-redo-data/logs-leg${LEG}/curveStudy-${cellId}.log" \
      | tail -1 > "/tmp/pushback-${cellId}.txt"
    node dist/bin/vp-dev.js research grade-reasoning \
      --issue "$issueId" --target-repo "$TARGET_REPO" \
      --decision pushback --pushback-path "/tmp/pushback-${cellId}.txt" \
      --out "feature-plans/curve-redo-data/scores-leg${LEG}/${cellId}-judge.json"
  fi

  rm -rf "$cellClone"
done
```

(Leg 2: same script with `LEG=2`, `TARGET_REPO=szhygulin/vaultpilot-development-agents`, framework lookup is per-issue but for leg 2 always `node-test`.)

## Cost gates + safety

- **Test generation** (Phase 2 prep): 13 issues × ~$1.20 = ~$15. Run once; tests committed to repo.
- **Per-cell budget**: `VP_DEV_MAX_COST_USD=10` env var caps each individual cell.
- **Per-leg budget**: each leg burns ~$130–$150 cells + ~$150 judge. Hard cap: `--max-total-cost-usd 1500` per leg.
- **Smoke first**: dispatch 2 cells (smallest + largest trim, on one issue) before the full leg. Inspect the diff capture, then the score JSONs, then the cell-level quality. Operator gate before the full grid.

## Notes on harness state

- Hidden tests are committed in this PR for reproducibility (~500 KB total). Generated by `vp-dev research generate-tests` (Phase 1b). When the generator emits sibling imports (`./<x>.js`) matching the codebase's source-tree colocation pattern, the corpus's `testsDestRelDir` field pins where testRunner places them so resolution succeeds (e.g. `src/agent`, `src/state`); leg-1 issues + leg-2's #157 use the default `curve-redo-hidden-tests/` sandbox because their tests use only stdlib + `../src/...` paths.
- Cell logs / diffs / score JSONs are gitignored at runtime — each leg ships its results as a single tarball on its feature branch. Mirrors the `issue-179-leg2-bundle/leg-1-logs.tar.gz` pattern.
- Curve-study isolation (no target-repo CLAUDE.md, no user-global) is hardcoded per [#216](https://github.com/szhygulin/vaultpilot-development-agents/pull/216).
- The 18 trim agents are reused from `issue-179-leg2-bundle/` — same parent (`agent-916a`), same seeds, same per-trim CLAUDE.md content. Direct comparability with the K=13 baseline.
- `--issue-body-only` is required for closed-issue dispatches so the resolution-PR link in close comments doesn't contaminate the agent's task.
